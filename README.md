# Сервис микроблогов (Twitter-clone)

Бэкенд корпоративного сервиса микроблогов на **FastAPI** + **PostgreSQL**.
Пользователи могут публиковать твиты (с картинками), лайкать, подписываться
друг на друга и получать ленту из твитов тех, на кого они подписаны,
отсортированную по популярности.

Проект полностью контейнеризирован и поднимается одной командой через Docker
Compose. Вместе с API раздаётся и готовый фронтенд (Vue).

## Возможности

- Создание и удаление собственных твитов
- Прикрепление картинок к твитам
- Лайки / снятие лайков
- Подписка / отписка на других пользователей
- Лента из твитов от тех, на кого подписан пользователь (и своих),
  отсортированная по числу лайков
- Профиль пользователя со списками подписок и подписчиков
- Документация API (Swagger) доступна сразу после запуска

## Технологии

- **Python 3.11**, **FastAPI**
- **SQLAlchemy 2.0** (async) + **asyncpg**
- **PostgreSQL 15**
- **Docker / Docker Compose**
- Тесты: **pytest** (+ aiosqlite для тестовой БД)
- Качество кода: **black**, **isort**, **flake8**, **mypy**

## Аутентификация

Регистрация не предусмотрена (это корпоративная сеть — пользователи заводятся
заранее). Все запросы, изменяющие данные, требуют HTTP-заголовок:

```
api-key: <ключ_пользователя>
```

Фронтенд подставляет ключ автоматически. Для демонстрации в базе уже создаются
два пользователя:

| api-key | имя   |
|---------|-------|
| `test`  | user  |
| `test2` | alice |

Они взаимно подписаны друг на друга, и у них есть несколько демо-твитов, чтобы
лента не была пустой.

## Запуск

Требуется установленный **Docker** и **Docker Compose**.

```bash
docker-compose up -d
```

После старта будут доступны:

| Что | Адрес |
|-----|-------|
| Веб-интерфейс | http://localhost:8000 |
| Документация API (Swagger) | http://localhost:8000/docs |
| Проверка состояния | http://localhost:8000/health |

Остановить:

```bash
docker-compose down
```

Остановить и удалить данные (БД + загруженные картинки):

```bash
docker-compose down -v
```

### Переменные окружения

Подключение к базе задаётся через `DATABASE_URL` (уже прописана в
`docker-compose.yaml`):

```
postgresql+asyncpg://admin:admin@db:5432/clon_twitter_db
```

## Эндпоинты API

Все ответы имеют поле `result` (`true`/`false`). Ошибки возвращаются в формате
`{"result": false, "error_type": "...", "error_message": "..."}`.

### Твиты
| Метод | Путь | Описание |
|-------|------|----------|
| `GET` | `/api/tweets` | Лента твитов |
| `POST` | `/api/tweets` | Создать твит |
| `GET` | `/api/tweets/{id}` | Получить твит по id |
| `DELETE` | `/api/tweets/{id}` | Удалить свой твит |

### Лайки
| Метод | Путь | Описание |
|-------|------|----------|
| `POST` | `/api/tweets/{id}/likes` | Лайкнуть твит |
| `DELETE` | `/api/tweets/{id}/likes` | Убрать лайк |

### Подписки
| Метод | Путь | Описание |
|-------|------|----------|
| `POST` | `/api/users/{id}/follow` | Подписаться |
| `DELETE` | `/api/users/{id}/follow` | Отписаться |

### Пользователи
| Метод | Путь | Описание |
|-------|------|----------|
| `GET` | `/api/users/me` | Свой профиль |
| `GET` | `/api/users/{id}` | Профиль по id |

### Медиа
| Метод | Путь | Описание |
|-------|------|----------|
| `POST` | `/api/medias` | Загрузить картинку (multipart, поле `file`) |

Полное описание полей запросов и ответов — в Swagger (`/docs`).

## Разработка

Зависимости для разработки (линтеры, тесты) — в `requirements-dev.txt`.

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements-dev.txt
```

### Тесты

```bash
pytest
```

С отчётом о покрытии:

```bash
pytest --cov=app
```

### Проверка качества кода

```bash
isort app/ tests/
black app/ tests/
flake8 app/ tests/
mypy app/
```

## Структура проекта

```
app/
├── main.py            # точка входа, обработчики ошибок, раздача статики
├── database.py        # подключение к БД, инициализация, сидинг
├── models/            # модели SQLAlchemy (User, Post, Like, Follow, Media)
├── schemas/           # схемы Pydantic (валидация запросов/ответов)
├── routers/           # эндпоинты (tweets, likes, users)
├── services/          # бизнес-логика
└── middleware/        # аутентификация по api-key
dist/                  # собранный фронтенд (Vue)
tests/                 # тесты
```
